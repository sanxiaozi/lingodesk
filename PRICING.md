# LingoDesk 定价方案

> 定价原则:体验丝滑优先。支付走 Telegram Stars 原生订阅(用户在 @LingoDeskbot 里两下点完,自动续费)。

## 两档

| | **Free(永久免费)** | **Pro — 500 ⭐ / 月** |
|---|---|---|
| 每月翻译回复(出站) | 300 条 | ♾️ 无限 |
| 联系人 | 无限 | 无限 |
| 全部功能(双语卡片/预览发送/媒体存档/主动发起) | ✅ | ✅ |
| 优先支持 | — | ✅ |

- **500 Stars ≈ $7**(用户手机内购约付 $6.99,桌面约 $6.5);平台到手约 $4.5–6.5(App Store 抽成后)。
- 翻译成本极低(Claude Haiku 一条约 $0.001),Pro 毛利健康。

## 关键设计

- **收费维度 = 每月出站翻译条数,不是联系人数**。核心是不伤信任:入站永远翻译、永远建话题、**永不丢客户消息**;额度只影响「你主动点发送给客户」的翻译回复。超额后入站照常,只是出站暂停,下月 1 号重置或订阅解锁。
- **免费 300 条/月**:够轻度用户跟一两个人聊一整月;真做生意的人几天就超 → 自然转化。
- **只支持月付**:Telegram Star 订阅唯一允许 30 天周期(无年付/周付)。年付以后要做得走网页 Stripe。
- **收款集中在官方 @LingoDeskbot**:多租户各用自己的 bot,但订阅必须在官方门户里付(否则用户等于付给自己)。租户在自己控制台超额 → 引导跳 @LingoDeskbot 订阅 → 付款进平台 → 更新该租户 plan → 控制台解锁。
- **自托管不计费**:`BILLING_ENABLED` 开关,官方云托管设 1,开源自托管默认关(所有人无限)。

## 触达上限的处理(丝滑关键)

免费用户额度用完后,点「发送给客户」不粗暴切断:弹一次温和提示「本月免费额度已用完,⭐ 升级 Pro 解锁无限」+ 一键跳转订阅按钮,**草稿保留**;升级后回来重新点发送即可。`/status` 全程显示「本月出站 X/300」。

## 相关命令

- 用户:`/subscribe` 订阅 Pro · `/status` 查套餐用量 · `/paysupport` 支付支持(合规必需)
- 管理员:`/grant <id> [月数]` 授予 Pro(种子用户/运营方免费)· `/revoke <id>` 降级

## 早期策略

内测老用户 + 种子用户用 `/grant` 给「早期用户免费 N 个月」,作为踩坑试用的回报与口碑来源。先低价起量冲 100 订阅用户,后续有数据再优化定价(老用户可锁价)。
