快速上手:從零到跑通,一步不漏

照著做即可,全程約 15 分鐘。每一步都標註了在哪個介面點什麼,以及最容易踩的雷。

目錄 準備清單 第 1 步 · 在 BotFather 建立 bot(含兩個關鍵開關) 第 2 步 · 在 Telegram Business 裡綁定 bot 第 3 步 · 建立控制台論壇群 第 4 步 · 取得三個 ID / Key 第 5 步 · 填寫 .env 第 6 步 · 啟動 第 7 步 · 試聊驗證 常見問題排解

🧾 準備清單

  • Telegram Premium 會員 —— Telegram Business 功能(訊息回灌給 bot)是 Premium 專屬,沒有 Premium 整套流程無法運作。
  • Node.js 20 或更高 —— 終端機跑 node -v 確認;沒有就去 nodejs.org 裝 LTS 版。
  • Anthropic API Key —— 翻譯引擎。第 4 步教你申請;可選再備一個 OpenAI Key 做備援。
  • LingoDesk 程式碼 —— 免費開源,託管在 GitHub,第 4b 步一條指令複製。
  • 一台常開的電腦 —— bot 程序要一直跑著才能收發(後面可遷到伺服器)。

1在 BotFather 建立 bot

在 Telegram 搜尋 @BotFather(認明官方藍 V),對話如下:

🤖 BotFather
/newbot
BotFatherAlright, a new bot. How are we going to call it? Please choose a name for your bot.
LingoDesk 助手
BotFatherGood. Now let's choose a username for your bot. It must end in bot.
my_lingodesk_bot
BotFatherDone! Use this token to access the HTTP API:
8123456789:AAHfK3xxxxxxxxxxxxxxxxxxxxxxxxx
Keep your token secure and store it safely…
  • 「名字」隨意,是顯示名稱;「username」必須全域唯一且以 bot 結尾,被佔用就換一個。
  • 最後回覆裡那串 數字:字母 就是 BOT_TOKEN,先複製存好。它等於 bot 的密碼,別發給任何人。

1a. 開啟 Business Mode(必做)

還是在 BotFather 裡:

/mybots→ @my_lingodesk_bot→ Bot Settings→ Business Mode→ Turn on
不開 Business Mode,第 2 步在 Telegram Business 的機器人列表裡根本搜尋不到你的 bot。這是最常見的卡關點。

1b. 關閉 Group Privacy(必做)

/mybots→ @my_lingodesk_bot→ Bot Settings→ Group Privacy→ Turn off
Privacy 預設開啟,開啟狀態下 bot 看不到你在控制台群裡打的中文——表現為「我在話題裡發訊息,bot 毫無反應」。務必關掉(BotFather 會顯示 Privacy mode is disabled 才算成功)。

2在 Telegram Business 裡綁定 bot

用手機版 Telegram(此設定目前只在手機端有):

⚙️ 設定→ 💼 Telegram Business→ 🤖 聊天機器人 Chatbots
  1. 在搜尋框輸入你的 bot username(如 my_lingodesk_bot),點選它。
  2. 權限:把「回覆訊息 / Reply to messages」打開——不開的話 bot 只能看不能替你發。
  3. 「哪些聊天」建議選 所有一對一聊天(All 1:1 Chats);不想讓 bot 看到的私人聯絡人,加到排除列表(Excluded Chats)即可。
綁定成功後,你私訊裡收到的每則訊息都會靜默「抄送」給 bot——對方完全無感知,你手機上的聊天體驗也沒有任何變化。

3建立控制台論壇群

這個群是你的工作台:每個客戶一個話題(Topic),雙語卡片都彈在這裡。

  1. 新建群組:Telegram 裡點新建 → 新建群組,成員就選你的 bot(搜 username 能搜到),群名隨意,例如「LingoDesk 控制台」。
  2. 開啟話題:進群 → 點群名進資料頁 → 編輯 → 找到「話題 Topics」→ 打開。群會自動升級為超級群,介面變成左側話題列表。
  3. 把 bot 提為管理員:資料頁 → 管理員 → 新增管理員 → 選你的 bot → 勾選「管理話題 Manage Topics」(其餘預設即可)→ 儲存。
bot 要為每個客戶自動建立話題,沒有管理員的「管理話題」權限就會失敗——表現為「客戶發訊息了,群裡卻什麼都不彈」。
這個群只有你和 bot,不用拉任何真人。想加合夥人一起盯也可以,TA 在話題裡打中文同樣能觸發翻譯發送。

4取得三個 ID / Key

4a. ANTHROPIC_API_KEY(翻譯引擎)

  1. 打開 console.anthropic.com,註冊/登入。
  2. 左側 API Keys → Create Key → 複製那串 sk-ant-…(只顯示一次,存好)。
  3. Billing 裡儲值幾美元即可——預設用 Haiku 模型,一則譯文成本約 $0.001,幾美元夠用很久。

4b. 先把程式碼跑起來第一步:裝相依套件 + 填 BOT_TOKEN

終端機進入 LingoDesk 目錄:

git clone https://github.com/sanxiaozi/lingodesk.git cd lingodesk npm install cp .env.example .env

用任意編輯器打開 .env,先只填一行:BOT_TOKEN=第 1 步拿到的 token,儲存。

4c. FORUM_CHAT_ID 和 OWNER_USER_ID:一條指令拿到

  1. 在控制台群裡隨便發一句(例如「測試」)。
  2. 用你自己的帳號私訊 bot(搜它 username),發一句 /start。
  3. 回到終端機:
npm run ids ── 群/会话(supergroup 且 -100 开头的那行 = FORUM_CHAT_ID)── -1002345678901 supergroup · LingoDesk 控制台 ── 用户(你自己那行 = OWNER_USER_ID)── 712345678 Jason @your_username
兩個高亮的數字就是要填的值。如果輸出是「(空)」,說明那兩則訊息還沒發,發完重跑;如果報 409,說明 bot 正在執行,先停掉再跑。

5填寫 .env

把 .env 填完整(其餘行保持預設即可):

# 必填四项 BOT_TOKEN=8123456789:AAHfK3xxxxxxxxxxxxxxxxxxxxxxxxx FORUM_CHAT_ID=-1002345678901 OWNER_USER_ID=712345678 ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxxxxxx # 可选:OpenAI 兜底(Claude 偶发故障时自动切换,建议填) OPENAI_API_KEY=sk-xxxxxxxx
  • FORUM_CHAT_ID 帶負號,整串照抄(包括 -100 前綴)。
  • OWNER_USER_ID 一定要填:bot 靠它區分「客戶發來的」和「你自己手動回的」,不填會把你的手動回覆當客戶訊息處理。

6啟動

npm run db:push 🚀 Your database is now in sync with your Prisma schema. npm run dev ──────────────────────────────────────── 🚀 LingoDesk 已启动,bot = @my_lingodesk_bot 收→译→回→译 跨语言沟通中继: · 客户私聊你真人号 → 自动译中 + 在控制台 Topic 弹双语卡片 · 你在 Topic 内打中文 → 预览确认 → 译客户语,以你名义发出 ────────────────────────────────────────

看到這個橫幅就說明跑起來了。第一次跑 db:push 是初始化本機資料庫,以後不用再跑。

同一個 bot token 全球只能有一個實例在跑。如果報 409 Conflict,說明還有另一個程序佔著(例如上次沒關乾淨),全部停掉再啟動。

7試聊驗證

找一個別的 Telegram 帳號(分身帳號或朋友),用外語私訊你的真人帳號(不是私訊 bot!),例如發一句西語「Hola, ¿cuánto cuesta?」。控制台群裡應該幾秒內彈出一個以對方命名的新話題:

📋 LingoDesk 控制台 · 話題:@amigo_es
my_lingodesk_bot🆕新客戶 @amigo_es(es)
原文:Hola, ¿cuánto cuesta?
🇨🇳 你好,多少錢?
─────
↳ 本話題內打中文,自動譯 es 以你名義發出
方案 A 是 100 美元,包含全部功能。
my_lingodesk_bot📤 譯文預覽(→es),確認後才發給客戶:

El paquete A cuesta 100 dólares e incluye todas las funciones.

(你的中文:方案 A 是 100 美元…)

[ ✅ 發送給客戶 ] [ ✏️ 取消 ]

點「✅ 發送給客戶」,對方會看到你的真人帳號用流利西語回覆了 TA。到這裡,整條鏈路就通了 🎉

  • 對方發圖片/檔案/語音 → 自動轉進話題並存檔。
  • 語言辨識錯了?在話題裡發 /lang es 手動修正,發 /help 看全部指令。
  • 你在手機上直接手動回覆客戶也沒問題,bot 會把它記錄進話題存檔。

🔧 常見問題排解

客戶發訊息了,控制台群什麼都不彈

按順序檢查:① BotFather 裡 Business Mode 是否 Turn on(第 1a 步);② Telegram Business → Chatbots 是否綁定了 bot、該聊天是否在涵蓋範圍內(第 2 步);③ bot 是否是群管理員且有「管理話題」權限(第 3 步);④ 終端機裡 bot 是否還在跑、有無報錯。

我在話題裡打中文,bot 沒反應

九成是 Group Privacy 沒關(第 1b 步)。去 BotFather → Bot Settings → Group Privacy → Turn off,然後把 bot 踢出群再重新拉進來(權限變更需要重進群才生效),重新設為管理員。

啟動報 409 Conflict

同一 token 有兩個實例在跑。找到並停掉舊程序(Ctrl+C 或 pkill -f "tsx.*main.ts"),等十秒再啟動。

點「發送給客戶」報錯:客戶太久沒往來

Telegram 限制:太久沒互動的對話,bot 不能替你主動發起(報 BUSINESS_PEER_USAGE_MISSING)。讓對方先隨便發一句啟用,或這一次你在手機上手動回。

發送報錯:Business 連線失效

去 設定 → Telegram Business → Chatbots 裡把 bot 移除再重新加入,或讓客戶發則新訊息——bot 會自動捕獲新的連線並恢復。

翻譯語言不對 / 想換目標語言

語言在客戶第一則訊息時鎖定(避免短訊息誤判來回跳)。在該客戶的話題裡發 /lang 語言代碼 即可修正,如 /lang pt(葡語)、/lang ar(阿語)。

bot 重啟後,之前的「發送/取消」按鈕點了沒反應

待確認的譯文預覽存在記憶體裡,重啟會失效(按鈕會提示)。在話題裡把中文重新打一遍即可。客戶對應和聊天記錄都在資料庫裡,不會丟。

🎉 跑通了?

現在起,客戶用任何語言私訊你,你只管打中文。遇到問題先看上面的排解清單,沒解決就寫信給我們,附上終端機報錯截圖。
